Log Analysis

Tools Used - Azure Sentinel

Vid not available

Step 1: In the Azure Portal. search for "Azure Sentinel", and set up a new workspace

Step 2: click on "Data connectors" and add data sources

Step 3: Click "connect" for each data source and configure the data connectors

Step 4: Go to "Logs" section, Use KQL (Kusto Query Language) to write simple queries to explore log data

Step 5: Go to "Workbooks" section, create a new workbook to build visualizations based on your log data

Step 6: Navigate to "Analytics" section, click on Rules and customize existing rules or create new rules

Step 7: Move to "Incidents" tab, set up incident rules, define incident types and workflows for incident management

Step 8: When there is an incident, head over to "Incidents" tab, select the incident and use the investigation tools to analyze and understand the situation

Step 9: You can also create automated responses and define actions to mitigate potential threats in the "Playbooks" section.
